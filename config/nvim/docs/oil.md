# oil.nvim

`oil.nvim` 是一个“把目录当成可编辑 buffer”的文件管理器。

和传统文件树不一样，它更像在编辑一个目录列表：
- 新建文件/目录：直接新增一行
- 重命名：直接改名字
- 删除：直接删掉那一行
- 最后用 `:w` 一次性应用改动

## 入口

- `-`
  在当前文件所在目录打开 `oil`
- `<leader>o`
  在当前窗口打开 `oil`
- `<leader>O`
  用浮窗打开 `oil`

`<leader>e` 仍然保留给你现有的文件树，不和 `oil` 冲突。

## 最常用操作

- `<CR>`
  打开光标所在文件/目录
- `-`
  返回上一级目录
- `g.`
  显示/隐藏隐藏文件
- `<C-s>`
  垂直分屏打开
- `<C-h>`
  水平分屏打开
- `<C-t>`
  在新 tab 打开
- `<C-p>`
  预览文件
- `gx`
  用系统默认程序打开
- `<C-l>`
  刷新
- `g?`
  查看当前所有按键
- `<C-c>`
  关闭 `oil`

## 编辑目录

这是 `oil` 最重要的用法。

1. 打开一个目录。
2. 像编辑文本一样修改列表。
3. 执行 `:w`。
4. `oil` 会把这些文本改动变成文件系统操作。

## 典型例子

### 新建文件

在目录 buffer 里新增一行：

```text
notes.md
```

然后执行：

```vim
:w
```

### 新建目录

新增一行：

```text
docs/
```

然后 `:w`。

### 重命名文件

把这一行：

```text
old_name.py
```

直接改成：

```text
new_name.py
```

然后 `:w`。

### 删除文件

把对应那一行删掉，然后 `:w`。

## 关键认知

- 在 `oil` 里做的改动，只有 `:w` 之后才真正生效。
- 如果改完不想执行，直接 `:q!` 退出。
- 它很适合批量重命名、整理目录结构。
- 它不是替代搜索器；找文件还是继续用 `telescope` 更合适。

## 你的当前配置

- 默认接管目录打开行为，所以 `nvim .` 或 `:e some_dir/` 会直接进入 `oil`
- 默认不显示隐藏文件，按 `g.` 切换
- 浮窗模式带圆角边框
- 使用 `nvim-web-devicons` 提供图标
