## Yanky.nvim

> 增强的剪贴板管理，支持历史记录、循环浏览、与 Telescope 集成

### 🎯 核心功能

1. **剪贴板历史记录** - 保存最近的 100 条复制记录
2. **循环浏览** - 像 IDE 一样循环粘贴历史
3. **持久化存储** - 使用 SQLite 保存历史，重启后仍然可用
4. **高亮反馈** - 复制和粘贴时短暂高亮

### ⌨️ 快捷键

#### 基本操作（增强版 yank/put）

| 快捷键 | 模式 | 功能 |
|--------|------|------|
| `y` | n, x | 复制（增强版） |
| `p` | n, x | 粘贴在光标后 |
| `P` | n, x | 粘贴在光标前 |
| `gp` | n, x | 粘贴并将光标移到文本后 |
| `gP` | n, x | 粘贴到光标前并将光标移到文本后 |

#### 历史循环

| 快捷键 | 模式 | 功能 |
|--------|------|------|
| `[p` | n | 下一个粘贴历史（**n**ext） |
| `]p` | n | 上一个粘贴历史（**p**revious） |
| `<leader>pp` | n | 上一个粘贴历史（备选） |
| `<leader>pn` | n | 下一个粘贴历史（备选） |

> 在粘贴后使用 `[p`/`]p` 可以替换为历史中的其他内容
> 
> 💡 为什么用 `p` 而不是 `y`？因为我们在操作的是 **粘贴（put/paste）历史**，`p` 更符合语义
> 💡 注意：`[p` = next（更新的历史），`]p` = previous（更旧的历史）

#### Telescope 集成

| 快捷键 | 功能 |
|--------|------|
| `<leader>fy` | 打开剪贴板历史（Telescope） |

#### 文本对象

| 快捷键 | 模式 | 功能 |
|--------|------|------|
| `iy` | o, x | 选择复制的文本内容 |
| `ay` | o, x | 选择复制的文本及其周围 |

### 📝 使用示例

#### 示例 1: 基本复制粘贴

```
yy          # 复制整行
p           # 粘贴（最新的历史）
[p          # 替换为历史中的下一条（next，更新的）
]p          # 替换为历史中的上一条（previous，更旧的）
```

#### 示例 2: 交换两行

```
# 第 10 行和第 20 行交换
:10         # 跳到第 10 行
yy          # 复制
:20         # 跳到第 20 行
"0P         # 粘贴（不覆盖默认寄存器）
dd          # 删除第 20 行（原内容）
:10         # 回到第 10 行
p           # 粘贴
```

#### 示例 3: 使用 Telescope 选择历史

```
<leader>fy  # 打开历史列表
# 使用 j/k 或 Ctrl-n/p 选择
# 按 <CR> 粘贴
```

#### 示例 4: 多光标粘贴

```
# 假设你已经复制了一些文本
# 在多个位置粘贴相同内容
p           # 第一次粘贴
.           # 重复上一次操作
# 或者
]p          # 循环到相同内容再粘贴
```

#### 示例 5: 复制到系统剪贴板

```
"+y         # 复制到系统剪贴板
"+p         # 从系统剪贴板粘贴
# 或者（自动同步开启时）
y           # 自动同步到系统剪贴板
p           # 自动从系统剪贴板粘贴
```

## Substitute.nvim

> 智能替换工具，支持 exchange 模式、范围替换

### 🎯 核心功能

1. **智能替换** - 用寄存器内容替换目标，但保留寄存器
2. **Exchange 模式** - 交换两个文本块的位置
3. **范围替换** - 在指定范围内批量替换
4. **高亮反馈** - 替换时短暂高亮

### ⌨️ 快捷键

#### 基本替换

| 快捷键 | 模式 | 功能 |
|--------|------|------|
| `s` + motion | n | 用默认寄存器替换 motion 范围 |
| `ss` | n | 替换整行 |
| `S` | n | 替换从光标到行尾 |
| `s` | x | 在 visual 模式下替换选区 |

#### Exchange 模式（交换）

| 快捷键 | 模式 | 功能 |
|--------|------|------|
| `sx` + motion | n | 标记第一个要交换的文本块 |
| `sx` + motion | n | 标记第二个文本块并执行交换 |
| `sxx` | n | 标记整行进行交换 |
| `sxc` | n | 取消交换操作 |
| `X` | x | 在 visual 模式下标记交换 |

#### 范围替换

| 快捷键 | 模式 | 功能 |
|--------|------|------|
| `<leader>sr` + motion | n | 在指定范围替换 |
| `<leader>srr` | n | 替换当前单词的所有出现 |
| `<leader>sr` | x | 在 visual 选区内替换 |

### 📝 使用示例

#### 示例 1: 基本替换

```vim
" 先复制一些内容
yiw          # 复制当前单词

" 然后替换其他内容
siw          # 替换光标下的单词为刚才复制的内容
si"          # 替换双引号内的内容
sap          # 替换整个段落
```

#### 示例 2: 替换整行

```vim
yy           # 复制一行
5j           # 向下移动 5 行
ss           # 用复制的行替换当前行
```

#### 示例 3: Exchange 交换两个单词

```vim
" 假设: foo bar
sxw          # 标记 foo（光标在 foo 上）
w            # 移动到 bar
sxw          # 标记 bar，自动交换
" 结果: bar foo
```

#### 示例 4: Exchange 交换两行

```vim
" 第 10 行和第 20 行交换
:10
sxx          # 标记第 10 行
:20
sxx          # 标记第 20 行，自动交换
```

#### 示例 5: 交换两个括号内的内容

```vim
" 代码: foo(aaa, bbb)
sxi(         # 标记 (aaa)
W            # 移动到 bbb
sxi(         # 标记 (bbb)，自动交换
" 结果: foo(bbb, aaa)
```

#### 示例 6: 范围替换（批量替换）

```vim
" 在当前函数内替换所有 foo 为 bar
<leader>sric
" 输入 foo
" 输入 bar
" 确认替换
```

#### 示例 7: Visual 模式下的替换

```vim
" 选中一些文本
viw          # 选中当前单词
" 复制其他内容后
" 选中要替换的文本
viwp         # 传统方式（会覆盖寄存器）
" 或者用 substitute
viws         # 用之前复制的替换，保留寄存器
```

#### 示例 8: 替换到行尾

```vim
" 复制一些内容
yy
" 替换从光标到行尾的部分
S            # 替换到行尾
```

## 组合使用技巧

### 🔥 高级技巧 1: 复制一次，粘贴多次

```vim
" 用 yanky 复制
yy           # 复制行

" 用 substitute 多次替换
3j
ss           # 替换此行
5j
ss           # 替换另一行
" 寄存器仍然保留原内容！
```

### 🔥 高级技巧 2: 交换 + 历史

```vim
" 交换两个文本块
sxx          # 标记第一行
5j
sxx          # 标记第二行，交换

" 不满意？用 yanky 撤销
u            # 撤销交换
]p           # 循环到交换前的内容
p            # 恢复
```

### 🔥 高级技巧 3: 批量格式化

```vim
" 复制完美格式化的代码块
<leader>fy   # 打开历史
" 选择需要的代码

" 用 substitute 批量替换其他位置的代码
<leader>sric # 在函数内替换
" 输入要替换的文本
" 回车确认
```

### 🔥 高级技巧 4: 函数参数重排

```vim
" 原代码: foo(arg1, arg2, arg3)
" 想重排为: foo(arg2, arg1, arg3)

f(           # 跳到括号
w            # 到 arg1
sxi,         # 标记 arg1（到逗号）
W            # 到 arg2
sxi,         # 标记 arg2，与 arg1 交换
" 完成！
```

### 🔥 高级技巧 5: 使用寄存器

```vim
" 复制到指定寄存器
"ayy         # 复制到寄存器 a

" 用指定寄存器替换
"asip        # 用寄存器 a 的内容替换段落
" 或者用 substitute
"ass          # 用寄存器 a 的内容替换行
```

### 🔥 高级技巧 6: 宏 + Yanky

```vim
" 录制宏
qa
yy           # 复制
j            # 下一行
ss           # 替换
q            # 结束录制

" 执行宏 10 次
10@a

" 不满意？撤销后用 yanky 循环历史
u
]p           # 循环历史
```

## 🎨 自定义配置

### Yanky 配置亮点

```lua
{
  ring = {
    history_length = 100,  -- 历史记录长度
    storage = "sqlite",    -- 持久化方式
  },
  highlight = {
    on_put = true,         -- 粘贴时高亮
    on_yank = true,        -- 复制时高亮
    timer = 500,           -- 高亮时间（毫秒）
  },
}
```

### Substitute 配置亮点

```lua
{
  highlight_substituted_text = {
    enabled = true,        -- 替换时高亮
    timer = 500,
  },
  exchange = {
    use_esc_to_cancel = true,  -- ESC 取消交换
  },
}
```

## 💡 提示

1. **为什么用 `s` 代替 `c`？**  
   `s` 在 substitute 中保留了寄存器内容，而 `c` 会覆盖。这样你可以复制一次，替换多次。

2. **Exchange 模式的魔力**  
   交换模式是 vim 中很难实现的操作，substitute 让它变得简单直观。

3. **Yanky 的持久化**  
   使用 SQLite 存储，重启 Neovim 后历史记录仍在。

4. **与 Telescope 集成**  
   使用 `<leader>fy` 可以模糊搜索历史，比循环更高效。
